Report A Change Once Prototypes

January 2026

Version three prototypes

Please note: The questions asked in the journey are based on the data capture requirements according to the latest versions of the relevant process maps / doc nodes. These maps are based on the as-is operational instructions. This may vary from actual processes currently in use.

All example content shown within the screens is based on GDS standards, style guidance and similar questions in other services. These are early drafts and will be iterated based on user evidence.

Change log

In version three we have:

  1. We have changed the name to ‘Tell DWP about a change’ to test a less specific name for comparison with the last round of UR (Tell DWP about a change of address / Report a change)
  2. We have iterated the start page to explain RaCO as the place to report general changes (not framed as change of address). We have also added new content to set expectations about other questions they may be asked, and added a Welsh translation link (requirement from private beta onwards)
  3. We have added an example knock out page ‘You cannot use this service’ for the radio option ‘my change is not listed’ on the change type page. This is to test whether this design helps users to understand what they can report in RaCO, and where to report other changes.
  4. We have iterated the tag labels to make the list and dependent tasks clearer. Research findings from the Nov 2025 round showed that some users thought they could complete the list in any order. We have added different tag labels like ‘cannot start yet’ based on GDS pattern to make this more explicit.
  5. We have iterated the address playback screen to make it clearer which address we are asking for. Research findings from the Nov 2025 round showed that users were confused by the design of this screen as they were unsure whether it referred to their previous or new address. We have based this iterated design on another example from Digital Channels that has tested well.
  6. We have iterated the evidence screens to:
    • create an example with screens per evidence type
    • create an example with one combined screen for all types
    • add realistic evidence types taken from the DWP evidence standards
    • add reassurance text to explain what the user should do if they do not have the evidence to hand or can’t upload it online. This example content is based on other similar evidence upload services including HMRC.
  7. We have iterated the confirmation page to:
    • add example content about notifications such as email or SMS
    • add example content to explain that citizens will be contacted if their claim needs to be reassessed
    • terated the example timeframe (6-8 weeks) to test a shorter timeframe (2-4 weeks). In the Nov 2025 UR round, users felt 6-8 weeks was a long time so we want to test perceptions around something shorter. Accurate average timeframes have not been determined yet, so these are just examples.

View version 3c - No longer eligible journey (ARCHIVE)

Version 3c - no longer eligible scenario: Citizen reports a change that will make them ineligible for the benefit. We agreed with the team 28/1/26 that we will park this as there is little difference from the other scenarios and limited value in testing at this stage.

This is because:

  • We cannot advise the user that they are not / may not be eligible - this must be a human decision. There are also potential F&E error implications to stating this within the service
  • RaCo will not touch policy
View version 3d - Save and return journey

Version 3d - save and return scenario: Citizen needs to leave the journey to find evidence and return later (save and return feature)

  • They do not have over £10k in savings
  • The money left over from the sale and their capital is less than £10K
  • Evidence requirement is a completion statement used to determine capital

Agent and appointee prototypes

View version 3e – save and return journey for agents

Version 3e - save and return scenario:A citizen (or appointee reporting a change on their behalf) is following the agent-led telephony journey and needs to provide evidence of this change:

  • Citizen has downsized in the UK
  • They owned and have sold their previous property. They used the proceeds to buy the new property with no mortgage, and have money left over from the sale
  • They do not have over £10k in savings
  • The money left over from the sale and their capital is less than £10K
  • Evidence requirement is a completion statement used to determine capital – the agent will give them options of how to provide this evidence (digital or postal)
  • Confirmation page explains next steps for agent to read to citizen / appointee

View version 3f – PDF output

Version 3f – PDF output for handover: An agent is provided with the data (personal details information and data captured in RaCo) to process a change of address where:

  • Citizen has downsized in the UK
  • They owned and have sold their previous property. They used the proceeds to buy the new property with no mortgage, and have money left over from the sale
  • They do not have over £10k in savings
  • The money left over from the sale and their capital is less than £10K
  • Citizen has provided a completion statement which the agent will use to determine capital

Note:This is based on current understanding of how the RaCO data will be handed to LoB agents to process changes. The purpose of this prototype is to test whether the data captured for this scenario would be sufficient for an agent to process this type of change.

November 2025

Version two prototypes

Change log

In version two we have:

  1. We have updated questions to include CA eligibility criteria e.g. full time education
  2. We have removed individual ‘check your answers’ per LoB
  3. The review and submit page has combined ‘check your answers’ listed for both CA and AA and one joint declaration (rather than a declaration per LOB)
  4. We have added a starting question ‘what do you need to change’ before task list in version 2b
  5. We have renamed the task list item form ‘change of address’ to ‘change details’
  6. We have changed the service name to ‘Tell DWP about a change of address’ in version 2a and ‘Report a change once’ in version 2b to test user research.

Work done prior to this includes:

Visualised user journey flows

  1. Link to visualised user journey flows exported to confluence
  2. Visualised user journey flows were been created to understand the path a user takes through RACO to complete a specific task, such as reporting their change of address.
  3. The user flows act as blueprints, mapping out every step, decision point, and screen from entry to goal completion, helping optimise for efficiency, reduce friction, and align stakeholders on user journeys
  4. These user journey flows were shared in the digital channels UCD monthly meet up call, useful contacts, insights and resources were shared from wider UCD team to support the RACO project
  5. User flows have also been presented back to the wider RACO/CADE team on regular calls.

About Report a Change Once

Report a Change Once lets people tell DWP about a change of circumstance once.

During alpha, we will test the change of address journey with citizens on a selection of benefits including:

  • Attendance Allowance
  • Carers Allowance
  • Disability Living Allowance
  • Pension Credit

You can find out more about the project on the Report a Change Once confluence page
